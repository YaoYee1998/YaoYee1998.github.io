Android studio配环境踩坑指南

####坑1 Andriod Studio报错提示：

Error:(26, 13) Failed to resolve: com.android.support:appcompat-v7:28.+

原因：Andriod Studio所使用的sdk版本与build.gradle配置文件中的配置不符，

通过sdk manager查看sdk版本为下图红色部分，即26.1.1

将 com.android.support:appcompat-v7:28.+ 中28 改为26，如下图，即可编译通过

关于Android Studio3.2新建项目无法运行出现Failed to find Build Tools revision 28.0.3的解决方法
问题：我创建好新项目后发现竟然无法运行，找不到运行按钮，而且出现Failed to find Build Tools revision 28.0.2错误如下图：

解决：去所创建项目下的app>build.gradle，打开build.gradle如下图

在compileSdkVersion 28下面加上buildToolsVersion “28.0.3”这一句，再去工具栏那里点一下和该文件一样的那个图标重新运行一下就可以了，如图

但是需要注意：buildToolsVersion “28.0.3”中的版本号对应的是在SDK Tools中下载的Android SDK Build-Tools的版本，路径是file>settings>Appearance&Behavior>System Settings>Android SDK>SDK Tools。如图
